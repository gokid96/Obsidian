Strangler Fig 패턴은 **모놀리식에서 MSA로 점진적으로 전환하는 방법**
AWS 등 주요 클라우드 벤더에서 권장하는 마이그레이션 패턴

---

## 핵심 아이디어

기존 모놀리식을 한번에 갈아엎지 않고, 기능 하나씩 떼어내서 MSA로 전환한다.

---

## 전환 과정

**1단계: 프록시 추가**
```
Client → Gateway → 모놀리식 (주문, 결제, 배송)
```

**2단계: 기능 하나씩 분리**
```
Client → Gateway → 모놀리식 (주문, 배송)
                → payment-service (분리됨)
```

**3단계: 점진적 이전**
```
Client → Gateway → 모놀리식 (배송만 남음)
                → payment-service
                → order-service
```

**4단계: 완전 전환**
```
Client → Gateway → payment-service
                → order-service
                → delivery-service
```

---

## 장점

| 장점 | 설명 |
|------|------|
| 리스크 낮음 | 점진적 전환 |
| 롤백 쉬움 | 문제 시 다시 모놀리식으로 라우팅 |
| 서비스 중단 없음 | 운영하면서 전환 가능 |

---

## 주의사항

- Gateway SPOF 가능 → 이중화 필요
- 모놀리식과 새 서비스 간 데이터 동기화 고려
- 전환 기간 동안 두 시스템 모두 유지보수
