Outbox + CDC는 **Outbox 패턴에 CDC를 결합**해서 스케줄러 없이 실시간 발행하는 방식

---

## 구조

```
Application → Outbox 테이블 저장 → Debezium 감지 → Kafka 발행
```

```
┌─────────────┐    ┌─────────────┐    ┌─────────┐
│ Application │ →  │ Outbox 테이블│ →  │Debezium │ → Kafka
│  (DB 저장)   │    │  (같은 DB)  │    │  (CDC)  │
└─────────────┘    └─────────────┘    └─────────┘
```

---

## 스케줄러 vs CDC

| | Outbox + 스케줄러 | Outbox + CDC |
|--|-------------------|--------------|
| 발행 방식 | 스케줄러 폴링 | 변경 감지 |
| 지연 | 폴링 주기 | 거의 실시간 |
| 추가 인프라 | 없음 | Debezium 필요 |

---

## 왜 조합하나?

- **CDC만 쓰면:** 비즈니스 테이블 직접 읽음 → 이벤트 포맷 제어 어려움
- **Outbox + CDC:** 원하는 이벤트 형태로 저장 → CDC가 그대로 발행

**실시간 + 이벤트 형태 제어** 둘 다 가능.
