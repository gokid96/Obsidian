## 체크 순서

1. **물리 계층** - 랜선 연결 확인, 링크 LED
2. **IP 확인** - `ip addr` 또는 `ifconfig`
3. **게이트웨이 ping** - `ping 192.168.0.1`
4. **DNS 확인** - `ping 8.8.8.8` vs `ping google.com`
5. **경로 추적** - `traceroute` 또는 `mtr`

## 주요 명령어

```bash
# IP 설정 확인
ip addr show

# 라우팅 테이블 확인
ip route show

# DNS 설정 확인
cat /etc/resolv.conf

# 포트 연결 테스트
nc -zv [호스트] [포트]
telnet [호스트] [포트]

# 네트워크 연결 상태
ss -tuln
netstat -tuln
```

## 문제 유형별 원인

|증상|가능한 원인|
|---|---|
|IP 없음|DHCP 실패, 케이블 문제|
|게이트웨이 ping 실패|IP 설정 오류, 스위치 문제|
|8.8.8.8 되고 도메인 안됨|DNS 문제|
|특정 포트만 안됨|방화벽, 서비스 다운|
