## 역할

- IP 주소에서 네트워크 부분과 호스트 부분 구분
- 같은 네트워크인지 판단

## 표기 방식

```
255.255.255.0 = /24 = 11111111.11111111.11111111.00000000
255.255.0.0   = /16 = 11111111.11111111.00000000.00000000
255.0.0.0     = /8  = 11111111.00000000.00000000.00000000
```

## 서브넷 마스크 계산

### /24 예시

```
IP:      192.168.1.100
Mask:    255.255.255.0 (/24)

네트워크 주소: 192.168.1.0   (AND 연산)
브로드캐스트:  192.168.1.255 (호스트 비트 모두 1)
사용 가능 IP:  192.168.1.1 ~ 192.168.1.254 (254개)
```

### /26 예시

```
/26 = 255.255.255.192 = 11111111.11111111.11111111.11000000

한 서브넷당 호스트: 2^6 - 2 = 62개

192.168.1.0/26:   192.168.1.0   ~ 192.168.1.63
192.168.1.64/26:  192.168.1.64  ~ 192.168.1.127
192.168.1.128/26: 192.168.1.128 ~ 192.168.1.191
192.168.1.192/26: 192.168.1.192 ~ 192.168.1.255
```

## 실무 계산 예시

### 문제 1: 호스트 50개 필요, 최소 서브넷은?

```
50개 호스트 → 최소 2^n - 2 ≥ 50
2^6 - 2 = 62 ≥ 50 ✓

/26 (64개 주소, 62개 호스트)
```

### 문제 2: 192.168.10.75/27의 네트워크 정보

```
/27 = 32개 주소 단위
75 ÷ 32 = 2.xx → 2번째 서브넷 (0부터 시작)
시작: 32 × 2 = 64

네트워크: 192.168.10.64
브로드캐스트: 192.168.10.95 (64+31)
범위: 192.168.10.65 ~ 192.168.10.94
```

### 문제 3: 두 IP가 같은 네트워크인가?

```
IP1: 192.168.5.130/25
IP2: 192.168.5.200/25

/25 = 128개 단위
130 → 128~255 구간 (두 번째 서브넷)
200 → 128~255 구간 (두 번째 서브넷)

같은 네트워크 ✓
```
