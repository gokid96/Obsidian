
## CIDR (Classless Inter-Domain Routing)

### 개념

- 클래스 없이 유연하게 네트워크 분할
- 표기: IP주소/프리픽스 길이
- 예: 192.168.1.0/24

## CIDR 계산 팁

```
/24: 256개 주소, 254개 호스트
/25: 128개 주소, 126개 호스트 (반으로)
/26: 64개 주소, 62개 호스트 (4등분)
...
프리픽스 1 증가 = 주소 개수 절반
```

## 서브네팅 (Subnetting)

하나의 네트워크를 여러 개로 분할

```
192.168.1.0/24를 4개로 나누기:
→ /26 (64개씩)

192.168.1.0/26   : 192.168.1.0 ~ 63
192.168.1.64/26  : 192.168.1.64 ~ 127
192.168.1.128/26 : 192.168.1.128 ~ 191
192.168.1.192/26 : 192.168.1.192 ~ 255
```

## 슈퍼넷팅 (Supernetting)

여러 네트워크를 하나로 합치기

```
192.168.0.0/24 + 192.168.1.0/24
= 192.168.0.0/23 (512개 주소)
```

## 명령어

```bash
# 서브넷 계산 도구
ipcalc 192.168.1.0/24
sipcalc 192.168.1.0/26
```

### ipcalc 출력 예시

```bash
$ ipcalc 192.168.1.0/26
Network:   192.168.1.0/26
Netmask:   255.255.255.192
Broadcast: 192.168.1.63
HostMin:   192.168.1.1
HostMax:   192.168.1.62
Hosts/Net: 62
```
