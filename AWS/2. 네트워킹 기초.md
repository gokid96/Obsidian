
### VPC (Virtual Private Cloud)
- AWS 안의 독립적인 가상 네트워크 (다른 고객과 완전 격리)
- 모든 EC2는 VPC 안에 존재
- IP 대역 설정 (예: 10.0.0.0/16)
- 자유로운 네트워크 설계 가능 (Subnet, 라우팅, 보안 규칙 등)
- 온프레미스 연결 가능 (VPN, Direct Connect)

### Subnet (서브넷)
- VPC를 용도별로 나눈 네트워크 구역
- VPC의 IP 대역을 더 작게 쪼갠 것 (예: 10.0.1.0/24, 10.0.2.0/24)
- **Public Subnet**: Internet Gateway 연결, 외부 접근 가능 (웹서버, jump-node)
- **Private Subnet**: 외부 직접 접근 차단, 내부망만 (DB, 백엔드 서버)
- 각 서브넷은 하나의 가용 영역(AZ)에 속함

### Internet Gateway (IGW)
- VPC와 인터넷을 연결하는 출입구
- Public Subnet에 연결되어 양방향 인터넷 통신 허용
- VPC당 1개만 연결 가능

### Route Table (라우팅 테이블)
- 네트워크 트래픽의 경로를 지정하는 규칙 집합
- 목적지 IP별로 게이트웨이 설정 (예: 0.0.0.0/0 → IGW)
- 각 서브넷마다 연결됨

### NAT Gateway
- Private 서브넷의 서버가 외부로 나가는 것만 허용 (apt update, 외부 API 호출)
- 외부에서 들어오는 연결은 차단 (보안)
- Public Subnet에 위치하고 탄력적 IP 필요

### IAM (Identity and Access Management)
- AWS 사용자/서비스의 권한 관리 시스템
- 정책(Policy)으로 세밀한 권한 제어 (누가, 무엇을, 어떻게)
- 최소 권한 원칙 적용 (필요한 것만 허용)

### VPN (Virtual Private Network)
- 온프레미스와 AWS를 암호화 터널로 연결
- 인터넷 경유 (느리지만 저렴, 시간당 $0.05)
- **Site-to-Site VPN**: 회사 네트워크 ↔ AWS VPC
- **Client VPN**: 개인 PC ↔ AWS VPC (재택근무)
