### VPC (Virtual Private Cloud) 
- AWS 안의 독립적인 가상 네트워크  (네트워크 격리, 자유로운 네트워크 설계,온프레미스 연결 가능) 
- 모든 EC2는 VPC 안에 존재 
- IP 대역 설정 (예: 10.0.0.0/16) 

### Subnet (서브넷)
- VPC를 용도별로 나눈 네트워크 구역 
- VPC의 IP 대역을 더 작게 쪼갠 것 (예: 10.0.1.0/24, 10.0.2.0/24) 
- **Public Subnet**: 인터넷 연결 가능 (Internet Gateway 연결, 웹서버/jump-node용) 
- **Private Subnet**: 내부망만 가능 (외부 직접 접근 차단, DB/백엔드 서버용) 
- 각 서브넷은 하나의 가용 영역(AZ)에 속함

- ### Internet Gateway (IGW) 
- VPC와 인터넷을 연결하는 출입구 
- Public Subnet에 연결 

- ### Route Table (라우팅 테이블) 
- 네트워크 트래픽 경로 지정 
- 목적지별 게이트웨이 설정 

- ### NAT Gateway - Private 서버의 외부 접속 허용 
- 들어오는 연결은 차단 (보안) 
- Public Subnet에 위치 

- ### IAM (Identity and Access Management) 
- AWS 사용자/서비스 권한 관리 
- 정책(Policy)으로 세밀한 권한 제어 
- 최소 권한 원칙 적용
### VPN
- 회사 네트워크 ↔ AWS 암호화 연결
- Site-to-Site VPN
- Client VPN
