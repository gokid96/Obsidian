### Load Balancer 세부
- **ALB**: HTTP/HTTPS, URL 라우팅
- **NLB**: TCP/UDP, 초고속 처리
- 헬스 체크, SSL 인증서 설정

### Route 53
- AWS DNS 서비스
- 도메인 → IP 변환
- 라우팅 정책 (지연 시간, 가중치 등)

### CloudWatch
- AWS 리소스 모니터링
- 메트릭 수집 (CPU, 메모리, 디스크)
- 알람 설정

## 네트워크 계층 구조
```
[인터넷]
    ↓
[Internet Gateway]
    ↓
┌─[VPC]──────────────────┐
│                         │
│ [Public Subnet]         │
│  ├─ jump-node          │
│  └─ NAT Gateway        │
│       ↓                │
│ [Private Subnet]       │
│  ├─ prod-1            │
│  └─ prod-2            │
│                         │
│ [Security Group]       │
│ [NACL]                 │
└─────────────────────────┘
```

## 보안 계층

1. **NACL** (서브넷 레벨) - 상태 비저장
2. **Security Group** (인스턴스 레벨) - 상태 저장
3. **IAM** (권한 관리)

## 비교 정리

### EBS vs S3
| | EBS | S3 |
|---|---|---|
| 연결 | 인스턴스 1개 전용 | 어디서든 접근 |
| 속도 | 빠름 (ms) | 느림 (수백ms) |
| 용도 | 실시간 읽기/쓰기 | 백업/보관 |
| 비용 | 비쌈 | 저렴 |

### Public vs Private Subnet
| | Public | Private |
|---|---|---|
| 인터넷 | 직접 연결 | NAT 경유 |
| 용도 | 웹서버, jump-node | DB, 백엔드 |
| 보안 | 낮음 | 높음 |

### Security Group vs NACL
| | Security Group | NACL |
|---|---|---|
| 레벨 | 인스턴스 | 서브넷 |
| 상태 | Stateful | Stateless |
| 규칙 | 허용만 | 허용/거부 |
