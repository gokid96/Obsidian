
## 비교표

|구분|MAC 주소|IP 주소|
|---|---|---|
|계층|L2 (데이터링크)|L3 (네트워크)|
|길이|48비트|32비트(IPv4), 128비트(IPv6)|
|표기|00:1A:2B:3C:4D:5E|192.168.1.1|
|할당|제조사 (HW 고정)|관리자/DHCP|
|범위|같은 네트워크 내|네트워크 간 라우팅|
|변경|원칙적 불변 (SW 변경 가능)|유동적|

## MAC 주소 구조

```
00:1A:2B:3C:4D:5E
└─┬─┘ └───┬───┘
 OUI    NIC 고유번호
(제조사)
```

- OUI (Organizationally Unique Identifier): 제조사 식별
- NIC: 네트워크 인터페이스 고유 번호

## ARP (Address Resolution Protocol)

- IP → MAC 주소 변환
- 같은 네트워크 내에서 동작

```
PC: "192.168.1.1의 MAC이 뭐야?" (브로드캐스트)
Gateway: "192.168.1.1은 00:11:22:33:44:55야"
```

## 명령어

```bash
# ARP 테이블 확인
arp -a
ip neigh show

# MAC 주소 확인
ip link show
ifconfig
```

## 왜 둘 다 필요한가?

- **IP 주소**: 목적지까지 경로 찾기 (라우팅)
- **MAC 주소**: 같은 네트워크 내 실제 전달

```
[PC] → [라우터1] → [라우터2] → [서버]

IP 주소: 처음부터 끝까지 동일
MAC 주소: 구간마다 변경됨
```
