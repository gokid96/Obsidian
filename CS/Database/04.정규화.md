## 3. 정규화

데이터 중복 줄이고 이상현상(Anomaly) 방지하려고 함

---

### 이상현상(Anomaly)

나쁜 예시 - 중복 있는 테이블:

|id|name|dept|dept_phone|
|---|---|---|---|
|1|철수|개발팀|1234|
|2|영희|개발팀|1234|
|3|민수|기획팀|5678|

- **삽입 이상**: 부서만 추가하고 싶은데 사람 없으면 못 넣음
- **수정 이상**: 개발팀 번호 바뀌면 여러 행 다 수정해야 함
- **삭제 이상**: 민수 지우면 기획팀 정보도 사라짐

---

### 1NF (제1정규형)

각 컬럼에 값 하나만 (원자값)

❌ 잘못된 예

|id|phone|
|---|---|
|1|010-1234, 010-5678|

⭕ 수정

|id|phone|
|---|---|
|1|010-1234|
|1|010-5678|
### 2NF (제2정규형) PK가 2개 이상일 때

1NF + 부분 종속 제거

❌ PK가 (학생id + 과목)인데:

|학생id|과목|학생이름|성적|
|---|---|---|---|

학생이름은 학생id만으로 알 수 있음 → 부분 종속

⭕ 테이블 분리

- 학생 (학생id, 학생이름)
- 수강 (학생id, 과목, 성적)

### 3NF (제3정규형)

2NF + 이행 종속 제거

❌ 잘못된 예

| 학생id | 학과  | 학과장 |
| ---- | --- | --- |

학생id → 학과 → 학과장

⭕ 테이블 분리

- 학생 (학생id, 학과)
- 학과 (학과, 학과장)
---
### 정리

| 단계  | 핵심       |
| --- | -------- |
| 1NF | 원자값      |
| 2NF | 부분 종속 제거 |
| 3NF | 이행 종속 제거 |
