- 테이블 나눠놨는데 합쳐서 봐야 할 때

예시 테이블

**user**

|id|name|
|---|---|
|1|철수|
|2|영희|
|3|민수|

**order**

|id|user_id|item|
|---|---|---|
|1|1|키보드|
|2|1|마우스|
|3|2|모니터|

---

**INNER JOIN** - 양쪽 다 있는 것만
```sql
```sql
SELECT user.name, order.item -- 두개의 테이블에서 컬럼을 가져옴
FROM user                    -- 유저 테이블을 기준
INNER JOIN order ON user.id = order.user_id; 
-- 유저의 id가 오더테이블에 있다면 테이블에 보여줌 (민수는 주문 없어서 안 나옴)   
```  
결과:

| name | item |
| ---- | ---- |
| 철수   | 키보드  |
| 철수   | 마우스  |
| 영희   | 모니터  |

**LEFT JOIN** - 왼쪽 테이블 전부 + 매칭되는 오른쪽
```sql
SELECT user.name, order.item -- 두 테이블에서 컬럼 가져옴
FROM user -- 왼쪽 테이블 (기준) 
LEFT JOIN order ON user.id = order.user_id; 
-- user 테이블은 전부 보여주고, 매칭되는 order가 있으면 붙이고 없으면 NULL
```

|name|item|
|---|---|
|철수|키보드|
|철수|마우스|
|영희|모니터|
|민수|NULL|

민수도 나옴 (주문 없으니까 NULL)

---

**RIGHT JOIN** - LEFT 반대

- 오른쪽 테이블 전부 + 매칭되는 왼쪽
- 실무에서 잘 안 씀 (LEFT로 순서 바꾸면 됨)

---
**FULL OUTER JOIN** - 양쪽 전부

- 매칭 안 되는 것도 NULL로 다 나옴
- MySQL은 지원 안 함 (UNION으로 해야 함)

---

**정리**

| JOIN  | 결과           |
| ----- | ------------ |
| INNER | 교집합          |
| LEFT  | 왼쪽 전부 + 교집합  |
| RIGHT | 오른쪽 전부 + 교집합 |
| FULL  | 합집합          |
