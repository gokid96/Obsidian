### 기존 DB → 벡터 DB 파이프라인

```
[기존 DB: PostgreSQL, MySQL 등]
    ↓
1. 수집 - SELECT로 텍스트 데이터 추출
    ↓
2. 분할 - 긴 문서는 500~1000토큰씩 청킹
    ↓
3. 임베딩 - 각 청크를 벡터로 변환
    ↓
4. 저장 - 벡터 DB에 (원본 + 벡터 + 메타데이터)
```

```
[벡터 DB 검색 결과]
"연차는 입사 1년 후 15일 부여됩니다..."
"연차 신청은 그룹웨어 > 근태관리에서..."
        ↓
[LLM에게 전달]
"아래 문서를 참고해서 질문에 답해줘:
 {검색된 문서들}
 
 질문: 연차 신청 어떻게 해요?"
        ↓
[LLM 답변 생성]
"연차는 그룹웨어의 근태관리 메뉴에서 신청하시면 됩니다.
 입사 1년 후 15일이 부여되며..."
```

LangChain/LlamaIndex는
전체 흐름(검색 → 프롬프트 조합 → LLM 호출)을 쉽게 연결해주는 프레임워크
