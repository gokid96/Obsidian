## 정의
출력의 확률 분포를 조절하는 파라미터

## 원리
- 다음 토큰 후보들의 확률이 있을 때
- Temperature가 낮으면: 높은 확률 토큰에 집중 (뾰족한 분포)
- Temperature가 높으면: 확률 분포가 평평해짐 (다양한 선택)

## 값에 따른 특성

| 값       | 특성                       | 적합한 작업     |
| ------- | ------------------------ | ---------- |
| 0       | 항상 같은 결과 (deterministic) | 사실 기반, 코딩  |
| 0.3~0.5 | 일관성 있으면서 약간의 변화          | 일반 대화, 요약  |
| 0.7~1.0 | 창의적, 다양한 출력              | 창작, 브레인스토밍 |
| >1.0    | 너무 랜덤, 품질 저하             | 거의 안 씀     |
## Temperature

| 작업     | 낮은 Temperature | 높은 Temperature |
| ------ | -------------- | -------------- |
| 코딩     | ✅ 정확한 코드       | ❌ 이상한 변형       |
| 수학     | ✅ 정답           | ❌ 엉뚱한 계산       |
| 소설 쓰기  | ❌ 뻔한 전개        | ✅ 창의적 전개       |
| 브레인스토밍 | ❌ 비슷한 아이디어만    | ✅ 다양한 아이디어     |
| 광고 카피  | ❌ 밋밋함          | ✅ 톡톡 튀는 문구     |


## 실무 팁
- 코딩/분석: 0
- 일반 챗봇: 0.3~0.7
- 창작: 0.8~1.0
- 헷갈리면 0.7로 시작해서 조절

## 관련 파라미터
- **top_p**: 누적 확률 기준으로 후보 제한
- **top_k**: 상위 k개 토큰만 고려
