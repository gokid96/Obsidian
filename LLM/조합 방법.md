
| 구분      | Function Calling | RAG          | Fine-tuning     |
| ------- | ---------------- | ------------ | --------------- |
| 역할      | 실시간 데이터 가져오기     | 지식/규칙 참조     | 모델 행동/스타일 학습    |
| 대상      | DB, API, 외부 시스템  | 문서, 매뉴얼, 가이드 | 학습 데이터셋 (Q&A 쌍) |
| 예시      | 재고 조회, 매출 집계     | 발주 기준, 업무 절차 | 특정 말투, 도메인 전문성  |
| 특징      | 정형 데이터, 숫자       | 비정형 텍스트, 노하우 | 모델 가중치 자체 변경    |
| 데이터 최신성 | 항상 최신            | 인덱싱 시점 기준    | 학습 시점에 고정       |
| 비용      | API 호출 비용        | 벡터DB 운영 비용   | GPU 학습 비용 (높음)  |
| 업데이트    | 즉시 반영            | 재인덱싱 필요      | 재학습 필요 (시간 소요)  |

**언제 무엇을 쓰나요?**

- **Function Calling**: "현재 A상품 재고가 몇 개야?" → 실시간 조회 필요
- **RAG**: "우리 회사 환불 정책이 뭐야?" → 사내 문서 검색
- **Fine-tuning**: "우리 브랜드 톤앤매너로 답변해줘" → 모델 자체가 그 스타일을 체화

실무에서는 이 세 가지를 조합해서 사용하는 경우가 많다. 예를 들어 Fine-tuning으로 도메인 전문성을 입히고, RAG로 최신 문서를 참조하며, Function Calling으로 실시간 데이터를 가져오는 식

## 세 기술의 관계

| 구분                   | 정확한 역할                                                |
| -------------------- | ----------------------------------------------------- |
| **Function Calling** | SQL뿐 아니라 **모든 외부 시스템 호출** (API, DB, 계산기, 날씨 등)        |
| **RAG**              | 문서 검색 → 규칙뿐 아니라 **비정형 지식 전체** (매뉴얼, FAQ, 노하우)         |
| **Fine-tuning**      | 정형 데이터가 아니라 **모델의 행동 방식 자체를 변경** (말투, 도메인 전문성, 특정 포맷) |
