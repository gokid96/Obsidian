캡슐화: 내부를 보호막으로 감싸서 외부로부터 보호
상속: 부모의 것을 자식이 물려받아 이어감
추상화: 여러 것에서 공통된 특징을 뽑아내어 일반화
다형성: 하나의 것이 여러 가지 형태를 가질 수 있는 성질


**캡슐화**: 데이터와 메서드를 하나로 묶고, 접근제어자로 외부의 직접 접근을 막는것. 예를들어 필드를 private으로 선언하고 게터/세터로만 접근 허용
```java
class Person {
    private String name;
    private int age;
    
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```

---

**상속**: extends를 통해 부모 클래스의 필드와 메서드를 자식클래스에서 물려받아 재사용하는것
```java
class Animal {
    String name;
    void eat() { System.out.println("먹는다"); }
}

class Dog extends Animal {
    void bark() { System.out.println("멍멍"); }
}
```

---

**추상화**: abstract나 interface로 공통기능의 틀만 정의하고 구체적인 구현은 숨기는것. 예를들어 "동물은 소리낸다"만 정의하고 어떻게 소리내는지는 각 동물클래스에서 구현
```java
abstract class Shape {
    abstract double getArea();
}

class Circle extends Shape {
    double radius;
    double getArea() { return 3.14 * radius * radius; }
}
```

---
**다형성**: 하나의 타입이나 메서드가 여러 형태로 동작하는것. 예를들어 부모타입으로 선언하고 자식객체를 담아서 호출하면 각 자식의 오버라이드된 메서드가 실행됨
```java
class Animal {
    void makeSound() { System.out.println("동물 소리"); }
}

class Dog extends Animal {
    void makeSound() { System.out.println("멍멍"); }  // 오버라이드
}

class Cat extends Animal {
    void makeSound() { System.out.println("야옹"); }
}

class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        
        a1.makeSound();  // "멍멍"
        a2.makeSound();  // "야옹" - 같은 타입, 다른 동작
    }
}
```
