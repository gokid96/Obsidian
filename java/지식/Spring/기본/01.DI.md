### new로 직접 생성
```java
@RestController
public class UserController {
    private UserService userService = new UserService();
}
```

**문제:**
- 테스트할 때 Mock으로 바꿀 수 없음
- UserService가 필요한 다른 객체들도 내가 직접 생성해야 함

### DI로 주입받기
```java
@RestController
public class UserController {
    private final UserService userService;
    
    public UserController(UserService userService) {
        this.userService = userService;
    }
}
```

**장점:**

- 테스트할 때 Mock 주입 가능
- Spring이 알아서 객체 만들어서 넣어줌

## 가장 큰 이유: 테스트

```java
// new로 생성하면
public class UserController {
    private UserService userService = new UserService();
    // 테스트 시 진짜 DB 연결하는 UserService가 생김 → 위험!
}

// DI 사용하면  
@Test
void test() {
    UserService 가짜 = mock(UserService.class);
    UserController controller = new UserController(가짜);
    // 가짜 객체로 테스트 가능!
}
```
