
## JPA 관련 
### @ManyToOne vs @OneToMany 
- Post → User: @ManyToOne (게시글이 Many, 작성자가 One) 
- User → Post: @OneToMany (역방향) 
- 보통 @ManyToOne만 사용 (단방향) 
### FetchType.LAZY 
- 연관된 엔티티를 즉시 로딩하지 않음 
- 실제 사용할 때 로딩 (N+1 문제 주의) 
### 실수했던 것들 
### 1. userId vs User 객체 
- ❌ private Long userId; 
- ✅ private User author; 
- 이유: JPA는 객체 지향. FK는 @JoinColumn으로 처리 
### 2. LocalTime vs LocalDateTime 
❌ private LocalTime createdAt; 
✅ private LocalDateTime createdAt; 
이유: 날짜+시간 필요. LocalTime은 시간만.
