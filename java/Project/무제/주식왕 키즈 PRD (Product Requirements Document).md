## 1. 제품 개요

### 1.1 제품명

**주식왕 키즈** (가칭)

### 1.2 제품 비전

초등학생이 한국 주식 모의투자를 통해 재미있게 금융을 배우고, 학습 성과를 실제 용돈으로 보상받는 게임형 금융교육 플랫폼

### 1.3 목표

- 초등학생에게 실질적인 금융 교육 제공
- 게이미피케이션을 통한 지속적인 학습 동기 부여
- 부모-자녀 간 경제 대화 활성화
- 건전한 투자 습관 형성

### 1.4 성공 지표 (KPI)

- **3개월**: 가입자 1만 명, DAU 3,000명, 용돈 송금 1,000건/월
- **6개월**: 가입자 10만 명, DAU 3만 명, 리텐션율 40%
- **12개월**: 가입자 30만 명, 유료 전환율 5%, 월 매출 1억 원

## 2. 타겟 사용자

### 2.1 Primary User (주 사용자)

**아동 (초등 3~~6학년, 만 9~~12세)**

- 경제/금융에 관심이 있는 학생
- 용돈을 받고 있는 학생
- 게임과 모바일 앱에 익숙한 학생

**사용자 페르소나 - "민수" (초등 4학년)**

- 게임을 좋아하고 유튜브를 자주 시청함
- 용돈 월 2만 원을 받고 있음
- 부모님이 주식 투자를 하는 것을 보고 관심이 생김
- 돈을 모으고 싶지만 어떻게 해야 할지 모름

### 2.2 Secondary User (부 사용자)

**부모 (30~40대)**

- 자녀에게 경제교육을 시키고 싶은 부모
- 합리적 소비 습관을 가르치고 싶은 부모
- IT 친화적이며 앱 사용에 거부감이 없음

**사용자 페르소나 - "지영" (38세, 직장인 엄마)**

- 초등학교 4학년 아들을 둔 워킹맘
- 자녀에게 용돈 교육을 하고 싶지만 방법을 모름
- 체계적인 금융교육 도구를 찾고 있음
- 자녀의 학습 과정을 모니터링하고 싶음

## 3. 핵심 기능 요구사항

### 3.1 모의투자 시스템

#### 3.1.1 초기 자금 시스템

**요구사항**:

- 신규 가입 시 가상 자금 100만 원 자동 지급
- 매일 출석 시 가상 자금 1만 원 지급
- 미션 완수 시 추가 가상 자금 지급

**기술 요구사항**:

- 출석 체크 시간: 매일 00시 기준 초기화
- 중복 지급 방지 로직 필수
- 가상 자금 거래 내역 로그 저장

#### 3.1.2 거래 가능 종목

**요구사항**:

- 코스피/코스닥 상장 주식 중 아동 친화적 기업 선별
    - 우선 종목: 삼성전자, LG전자, 현대차, 카카오, 네이버, 엔씨소프트, 하이브 등
    - 게임사, 엔터테인먼트, 식품, 완구 기업 우선
- 주요 ETF 포함 (KODEX 200, TIGER 미국S&P500 등)

**제약사항**:

- 레버리지/인버스 ETF 제외
- 우선주 제외
- 관리종목 제외

#### 3.1.3 실시간 데이터 연동

**요구사항**:

- 실제 주식 시세 반영 (15분 지연 데이터 허용)
- 장 개장 시간에만 거래 가능 (평일 09:00~15:30)
- 호가 시스템 적용 (5단계 호가)

**기술 요구사항**:

- 한국거래소 API 또는 증권사 Open API 연동
- Redis를 활용한 시세 캐싱
- 1분 단위 데이터 업데이트

#### 3.1.4 거래 기능

**요구사항**:

- 매수/매도 주문
- 지정가/시장가 주문 지원
- 보유 종목 현황 조회
- 거래 내역 조회
- 실시간 수익률 계산

**제약사항**:

- 신용거래 불가
- 미수 거래 불가
- 1회 최대 거래 금액: 보유 자금의 30%

### 3.2 게이미피케이션 시스템

#### 3.2.1 레벨 시스템

**요구사항**:

- 5단계 레벨 체계 구현
    - Lv.1-5: 주식 새싹 🌱
    - Lv.6-10: 투자 모종 🌿
    - Lv.11-15: 투자 나무 🌳
    - Lv.16-20: 투자 전문가 🏆
    - Lv.21+: 주식왕 👑

**레벨업 조건**:

- 거래 횟수, 수익률, 미션 완료 수, 학습 진도를 종합한 경험치 시스템
- 레벨업 시 축하 애니메이션 및 보상

#### 3.2.2 뱃지 시스템

**요구사항**:

- 최소 20개 이상의 뱃지 종류
- 주요 뱃지:
    - "첫 주식 매수" - 첫 거래 완료 시
    - "배당주 수집가" - 배당주 5개 이상 보유 시
    - "장기 투자자" - 30일 이상 동일 종목 보유 시
    - "뉴스 마스터" - 뉴스 100개 읽기
    - "수익 달성" - 수익률 5%, 10%, 20% 달성 시

**UI 요구사항**:

- 뱃지 컬렉션 화면
- 미획득 뱃지는 실루엣으로 표시
- 뱃지 획득 시 팝업 알림

#### 3.2.3 미션 시스템

**요구사항**:

**일일 미션** (매일 00시 초기화)

- 로그인하기 → 가상머니 +10,000원
- 오늘의 경제 뉴스 읽기 → 가상머니 +5,000원
- 1개 종목 분석하기 → 가상머니 +10,000원

**주간 미션** (매주 월요일 00시 초기화)

- 5일 연속 접속 → 가상머니 +50,000원
- 3개 종목 매수하기 → 가상머니 +30,000원
- 금융 퀴즈 10개 맞히기 → 가상머니 +40,000원

**특별 미션** (이벤트성)

- 시즌별 챌린지
- 한정 기간 보너스 미션

### 3.3 용돈 연동 시스템 ⭐

#### 3.3.1 목표 설정 (부모 앱)

**요구사항**:

- 부모가 다음 항목 설정 가능
    - 목표 수익률 (예: 10%)
    - 보상 금액 (예: 5,000원)
    - 월 최대 한도 (예: 20,000원)
    - 용돈 요청 가능 횟수 (예: 월 4회)

**UI 요구사항**:

- 간편한 슬라이더 UI로 설정
- 설정 내용 실시간 미리보기
- 저장 시 자녀 앱에 알림

#### 3.3.2 용돈 요청 프로세스

**요구사항**:

**Step 1: 목표 달성**

- 포트폴리오 수익률이 목표치 도달
- "용돈 요청" 버튼 활성화
- 애니메이션 효과로 강조

**Step 2: 아이가 요청**

- "용돈 요청" 버튼 클릭
- 요청 사유 선택 (선택사항)
- 부모 앱에 푸시 알림 발송

**Step 3: 부모 승인**

- 부모가 알림 확인
- 자녀의 투자 성과 확인
- 승인/거절 선택

**Step 4: 용돈 지급**

- 승인 시 실제 계좌로 송금
- 자녀 앱에 축하 메시지
- 거래 내역 기록

**기술 요구사항**:

- 토스페이먼츠 또는 나이스페이 간편송금 API 연동
- 송금 실패 시 재시도 로직
- 모든 거래 내역 암호화 저장

#### 3.3.3 용돈 요청 트리거 종류

**요구사항**: 부모가 선택 가능한 트리거 유형:

- 수익률 달성 (+5%, +10%, +20%)
- 연속 수익일 (5일, 10일 연속 플러스)
- 투자 미션 (배당주 5개 매수 등)
- 학습 완료 (금융교육 10강 수료)
- 장기 투자 (30일 이상 보유)
- 포트폴리오 분산 (5개 이상 종목 보유)

#### 3.3.4 부모 대시보드

**요구사항**:

- 자녀 포트폴리오 실시간 조회
- 보유 종목 및 수익률
- 거래 내역 (최근 30일)
- 투자 성향 분석 리포트
- 용돈 지급 내역 및 누적 금액
- 월간 학습 리포트
- 목표 달성률 그래프

**UI 요구사항**:

- 대시보드 형태의 한눈에 보기
- 차트 및 그래프 시각화
- 주간/월간 리포트 다운로드 (PDF)

#### 3.3.5 안전장치

**요구사항**:

- 월 최대 용돈 한도 강제 적용
- 용돈 요청 가능 횟수 제한
- 손실 시에도 학습 보상 제공 (과정 중시)
- "투자 일기" 작성 시 추가 보상
- 이상 거래 패턴 감지 및 알림

    - 뱃지 컬렉션

## 6. 기술 스택

### 6.1 프론트엔드

- **프레임워크**: React Native
- **상태관리**: Redux Toolkit
- **UI 라이브러리**: React Native Paper
- **차트**: Victory Native

### 6.2 백엔드

- **언어**: Node.js (Express.js)
- **데이터베이스**: PostgreSQL (사용자 데이터), Redis (시세 캐싱)
- **인증**: JWT, OAuth 2.0
- **API**: RESTful API

### 6.3 외부 연동

- **주식 데이터**: 한국거래소 API 또는 증권사 Open API
- **결제/송금**: 토스페이먼츠, 나이스페이
- **푸시 알림**: Firebase Cloud Messaging (FCM)
- **분석**: Google Analytics, Amplitude

### 6.4 인프라

- **클라우드**: AWS (EC2, RDS, S3, CloudFront)
- **CI/CD**: GitHub Actions
- **모니터링**: Sentry, CloudWatch

## 8. 위험 요소 및 대응 방안

### 8.1 법률/규제 리스크

**위험**: 사행성 논란, 청소년 보호법 위반 **대응**:

- 교육 목적 명확화 (사행성 아님)
- 과정 중심 보상 (손실 시에도 학습 보상)
- 법무법인 자문 확보
- 청소년 보호 정책 수립

### 8.2 기술 리스크

**위험**: API 장애, 서버 다운, 데이터 유실 **대응**:

- 이중화된 서버 구조
- 정기 백업 (일 1회)
- 장애 모니터링 및 알림 시스템
- DR(재해복구) 계획 수립

### 8.3 시장 리스크

**위험**: 경쟁사 등장, 수익화 실패 **대응**:

- 빠른 시장 선점 (First Mover Advantage)
- 커뮤니티 강화로 락인 효과
- 다양한 수익 모델 실험
- 지속적인 사용자 피드백 수렴

### 8.4 안전 리스크

**위험**: 아동 개인정보 유출, 커뮤니티 유해 콘텐츠 **대응**:

- 철저한 보안 체계
- AI 기반 유해 콘텐츠 필터링
- 24시간 신고 접수 및 처리
- 부모 모니터링 기능 강화
