### **현재 프로젝트: JPA + MVC (전통적 Layered Architecture)**
```
✅ 배우는 것:
- Spring Boot 기본
- JPA/Hibernate
- REST API 설계
- DTO 패턴

- 트랜잭션 관리
- 테스트 코드
```

## **1. 게시판 (Board/Question)** 📝

### **기본 CRUD**
- ✅ **생성** (POST) - 글쓰기
- ✅ **조회** (GET) - 목록, 상세보기
- ✅ **수정** (PUT/PATCH) - 글 수정
- ✅ **삭제** (DELETE) - 글 삭제

### **추가 기능**
- ✅ **페이징** - 목록을 10개씩 나눠서 보기
- ✅ **검색** - 제목/내용으로 검색
- 조회수 증가
- 추천/좋아요

**API 예시:**
```
GET    /api/questions              # 전체 목록 (페이징)
GET    /api/questions?page=1&size=10  # 페이징
GET    /api/questions?keyword=스프링   # 검색
GET    /api/questions/{id}         # 상세보기
POST   /api/questions              # 글쓰기
PUT    /api/questions/{id}         # 수정
DELETE /api/questions/{id}         # 삭제
```

## **2. 유저 (User)** 👤

### **기본 기능**
- ✅ **회원가입** (생성, POST)
- ✅ **로그인** (인증)
- ✅ **로그아웃**
- ✅ **정보 수정** (PUT/PATCH)
- ✅ **회원탈퇴** (삭제, DELETE)

### **추가 기능**
- 프로필 조회
- 비밀번호 변경
- 이메일 인증

**API 예시:**
```
POST   /api/users/signup           # 회원가입
POST   /api/users/login            # 로그인
POST   /api/users/logout           # 로그아웃
GET    /api/users/me               # 내 정보 조회
PUT    /api/users/me               # 내 정보 수정
DELETE /api/users/me               # 회원탈퇴
```

---

## **3. 댓글 (Comment/Answer)** 💬

### **기본 CRUD**
- ✅ **생성** (POST) - 댓글 작성
- ✅ **조회** (GET) - 댓글 목록 (특정 게시글의)
- ✅ **수정** (PUT/PATCH) - 댓글 수정
- ✅ **삭제** (DELETE) - 댓글 삭제

### **추가 기능**
- 대댓글 (답글)
- 추천/좋아요

**API 예시:**
```
GET    /api/questions/{questionId}/answers     # 특정 질문의 댓글 목록
POST   /api/questions/{questionId}/answers     # 댓글 작성
PUT    /api/answers/{id}                       # 댓글 수정
DELETE /api/answers/{id}                       # 댓글 삭제
```

## **전체 구조 정리**

### **최소 기능 (MVP)** ⭐
```
게시판
├── 글쓰기
├── 목록 조회 (페이징)
├── 상세보기
├── 수정
└── 삭제

유저
├── 회원가입
├── 로그인
└── 로그아웃

댓글
├── 작성
├── 목록 조회
├── 수정
└── 삭제
```

**Level 1:**
- 검색 기능
- 조회수

**Level 2:**
- 파일 첨부
- 좋아요/추천

**Level 3:**
- 대댓글
- 알림 기능
- 카테고리

## **실제 구현 순서 추천**

### **Phase 1: 기본 CRUD** (1-2주)
```
1. 게시판 - 생성/조회/수정/삭제
2. 유저 - 회원가입/로그인 (간단하게)
3. 댓글 - 생성/조회/수정/삭제
```

### **Phase 2: 필수 기능** (1주)
```
4. 페이징
5. 검색
6. 인증/권한 (내 글만 수정/삭제)
```

### **Phase 3: 추가 기능** (선택)
```
7. 파일 업로드
8. 좋아요
9. 조회수
```

## **DB 관계**
```
User (유저)
├── 1 : N → Question (게시글 작성)
└── 1 : N → Answer (댓글 작성)

Question (게시글)
├── N : 1 → User (작성자)
└── 1 : N → Answer (댓글들)

Answer (댓글)
├── N : 1 → User (작성자)
└── N : 1 → Question (어떤 글의 댓글인지)
```
