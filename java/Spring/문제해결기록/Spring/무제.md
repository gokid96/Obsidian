### ✅ Step 3: 5개 문제 선택 & 해결 (각 1-2주)

**추천 순서:**

#### 🎯 문제 1: N+1 문제 해결
```
현상: 게시글 목록 조회 시 작성자 정보 때문에 쿼리 100개 실행
해결: Fetch Join 적용
측정: 쿼리 로그, 응답 시간 Before/After
블로그: "N+1 문제 발견과 해결 과정"
```

#### 🎯 문제 2: 조회수 중복 방지
```
현상: 새로고침마다 조회수 증가
해결 1차: 쿠키/세션으로 IP 체크
해결 2차: Redis Set으로 개선
블로그: "조회수 중복 방지 전략 비교"
```

#### 🎯 문제 3: JWT 인증 개선
```
현상: Access Token만 있어서 탈취 위험
해결: Refresh Token 도입
추가: Redis에 Refresh Token 저장
블로그: "JWT Refresh Token 구현기"
```

#### 🎯 문제 4: 동시성 제어 (좋아요 기능)
```
기능 추가: 게시글 좋아요 기능
현상: 동시 요청 시 카운트 꼬임
해결: 낙관적 락 vs 비관적 락 테스트
블로그: "좋아요 기능의 동시성 제어"
```

#### 🎯 문제 5: 쿼리 최적화 (커버링 인덱스)
```
현상: 게시글 목록 조회 느림
분석: EXPLAIN으로 실행 계획 확인
해결: 인덱스 추가 + 커버링 인덱스 적용
블로그: "MySQL 인덱스 최적화 실전"
```
