# Dirty Checking 주의사항

## 성능 관련 주의점
- 대량 데이터 처리 시 성능 저하 가능성
- 불필요한 UPDATE 쿼리 발생 주의
- 복잡한 로직은 명시적 save() 고려

## 안티 패턴 예시
```java
// 비추천: 불필요한 UPDATE 발생
user.setName(user.getName());

// 추천: 실제 변경 시에만 수정
if (!Objects.equals(currentName, newName)) {
    user.setName(newName);
}
```

## 주요 주의사항
1. 트랜잭션 범위 주의
2. 불필요한 setter 호출 최소화
3. 복잡한 업데이트는 벌크 연산 고려
