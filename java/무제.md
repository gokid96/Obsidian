캡슐화는 관련된 데이터와 기능을 하나의 캡슐, 즉 객체로 묶고 외부의 직접적인 접근을 막는 것 

추상화는 객체의 복잡한 내부를 숨기고 사용에 필요한 핵심 기능만 노출하는 것 

상속은 부모 클래스의 특징을 자식 클래스가 물려받아 코드를 재사용하고 확장하는 개념

마지막으로 다형성은 하나의 이름이나 형태가 상황에 따라 다른 역할을 수행하는 것을 의미합니다. 예를 들어 '소리내다'라는 기능이 동물 객체의 종류에 따라 '멍멍', '야옹' 등 다르게 동작하는 것이죠. 레고 블록에 비유할 수 있어요. 각 블록은 자신만의 모양과 기능을 가진 객체(캡슐화, 추상화)이고, 기본 블록을 바탕으로 더 특별한 모양의 블록을 만들 수 있으며(상속), 어떤 블록이든 다른 블록과 연결될 수 있죠(다형성).


객체지향 4대 특성 요약:

1. 캡슐화

- 데이터와 기능 은닉
- private, protected 접근제어자
- 정보 보호

2. 추상화

- 복잡성 숨기기
- 핵심 인터페이스 노출
- 사용자 관점 단순화

3. 상속

- 코드 재사용
- 계층적 구조
- 공통 특성 물려받기

4. 다형성

- 같은 메서드명, 다른 동작
- 유연한 객체 설계
- 확장성

``` java 
class Animal {
    void makeSound() {
        System.out.println("소리");
    }
}

class Dog extends Animal { // extends 상속 Override: 구현 재정의를 통한 다형성
    @Override                   
    void makeSound() { 
        System.out.println("멍멍");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("야옹");
    }
}
```
