## 객체 지향 프로그래밍 (OOP)
- **Object-Oriented Programming**
- 현실 세계를 **객체(Object)**로 모델링
- 객체들의 상호작용으로 프로그램 구성

### OOP의 4대 특성
1. **캡슐화** (Encapsulation)
2. **상속** (Inheritance)
3. **다형성** (Polymorphism)
4. **추상화** (Abstraction)

---

## 클래스 (Class)

### 클래스란?
- 객체를 만들기 위한 **설계도(틀)**
- **속성(필드)**과 **동작(메서드)**를 정의
- 사용자 정의 타입

### 기본 구조
```java
public class 클래스명 {
    // 필드 (속성)
    타입 필드명;
    
    // 생성자
    클래스명() { }
    
    // 메서드 (동작)
    반환타입 메서드명() { }
}
```

### 예시: 자동차 클래스
```java
public class Car {
    // 필드
    String brand;
    String color;
    int speed;
    
    // 메서드
    void accelerate() {
        speed += 10;
    }
    
    void brake() {
        speed -= 10;
    }
    
    void showInfo() {
        System.out.println(brand + " " + color + " " + speed + "km/h");
    }
}
```

## 객체 (Object)

### 객체란?
- 클래스를 기반으로 **실제로 생성된 실체**
- 클래스의 **인스턴스(Instance)**
- 메모리에 할당됨

### 객체 생성
```java
클래스명 변수명 = new 클래스명();
```

```java
Car myCar = new Car();
Car yourCar = new Car();
```

### 객체 사용
```java
// 필드 접근
myCar.brand = "현대";
myCar.color = "검정";
myCar.speed = 0;

// 메서드 호출
myCar.accelerate();
myCar.showInfo();  // 현대 검정 10km/h
```

## 클래스 vs 객체

| 클래스 | 객체 |
|--------|------|
| 설계도 | 실제 제품 |
| 틀, 템플릿 | 인스턴스 |
| 메모리 X | 메모리 O (Heap) |
| 하나만 존재 | 여러 개 생성 가능 |

```java
// 클래스: 붕어빵 틀
class 붕어빵틀 { }

// 객체: 실제 붕어빵들
붕어빵틀 붕어빵1 = new 붕어빵틀();
붕어빵틀 붕어빵2 = new 붕어빵틀();
붕어빵틀 붕어빵3 = new 붕어빵틀();
```

## 객체 생성 과정

```java
Car myCar = new Car();
```

### 1. 선언
```java
Car myCar;  // Stack에 참조 변수 생성 (null)
```

### 2. 생성 (new)
```java
new Car();  // Heap에 객체 생성
```

### 3. 할당
```java
myCar = new Car();  // 참조 변수에 주소 저장
```

### 메모리 구조
```
[Stack]              [Heap]
┌──────────┐         ┌─────────────────┐
│  myCar   │────────→│  Car 객체       │
│  @100    │         │  brand: null    │
└──────────┘         │  color: null    │
                     │  speed: 0       │
                     └─────────────────┘
```

## 하나의 파일에 여러 클래스

### 규칙
- **public 클래스**는 파일당 **하나만**
- public 클래스명 = 파일명
- public 아닌 클래스는 여러 개 가능

```java
// Car.java
public class Car {  // 파일명과 동일
    // ...
}

class Engine {  // public 없음 - 같은 파일에 작성 가능
    // ...
}

class Wheel {
    // ...
}
```

## 실전 예제

### 학생 클래스
```java
public class Student {
    // 필드
    String name;
    int age;
    String major;
    double gpa;
    
    // 메서드
    void study() {
        System.out.println(name + "이(가) 공부합니다.");
    }
    
    void introduce() {
        System.out.println("이름: " + name);
        System.out.println("나이: " + age);
        System.out.println("전공: " + major);
        System.out.println("학점: " + gpa);
    }
}
```

### 사용
```java
public class Main {
    public static void main(String[] args) {
        // 객체 생성
        Student s1 = new Student();
        
        // 필드 설정
        s1.name = "홍길동";
        s1.age = 20;
        s1.major = "컴퓨터공학";
        s1.gpa = 4.0;
        
        // 메서드 호출
        s1.introduce();
        s1.study();
    }
}
```

### 은행 계좌 클래스
```java
public class BankAccount {
    String accountNumber;
    String owner;
    int balance;
    
    void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println(amount + "원 입금. 잔액: " + balance);
        }
    }
    
    void withdraw(int amount) {
        if (amount > 0 && balance >= amount) {
            balance -= amount;
            System.out.println(amount + "원 출금. 잔액: " + balance);
        } else {
            System.out.println("출금 불가");
        }
    }
    
    void showBalance() {
        System.out.println(owner + "님의 잔액: " + balance + "원");
    }
}
```

## 객체 배열

```java
// 객체 배열 선언 및 생성
Student[] students = new Student[3];

// 각 요소에 객체 생성
students[0] = new Student();
students[1] = new Student();
students[2] = new Student();

// 값 설정
students[0].name = "홍길동";
students[1].name = "김철수";
students[2].name = "이영희";

// 순회
for (Student s : students) {
    System.out.println(s.name);
}
```

---

> [!tip] 핵심 정리
> - **클래스**: 객체의 설계도 (필드 + 메서드)
> - **객체**: 클래스로 생성된 실체 (인스턴스)
> - 생성: `new 클래스명()`
> - 접근: `객체.필드`, `객체.메서드()`
> - 객체는 **Heap 메모리**에 생성됨
