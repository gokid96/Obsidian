

```
┌─────────────┐
│    사용자    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────┐
│       Cloudflare        │
│     (DNS Only 모드)      │
├─────────────────────────┤
│ untitles.net → 76.76.21.21 (Vercel)
│ api.untitles.net → 3.37.191.88 (AWS)
└──────┬──────────────────┘
       │
       ▼
┌───────────────┬───────────────┐
│    Vercel     │  Backend API  │
│  (프론트엔드)  │   (AWS 서울)   │
└───────────────┴───────────────┘

```
- `untitles.net` 접속 → Cloudflare DNS → Vercel IP 반환 → Vercel 연결
- `api.untitles.net` 호출 → Cloudflare DNS → AWS IP 반환 → AWS 연결

DNS에서 도메인별로 각각 IP를 알려주고, 사용자 브라우저가 직접 해당 서버로 연결
```
- 사용자가 `untitles.net` 접속 → Vercel에서 프론트엔드 페이지 받아옴
- 로그인 버튼 클릭 → 프론트엔드가 `api.untitles.net`으로 API 요청
- 브라우저가 `api.untitles.net` DNS 조회 → AWS IP 받아서 연결
- 백엔드에서 응답 받아서 화면에 반영
```

이미지 업로드 추가후 
```
┌─────────────┐
│    사용자    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────┐
│           Cloudflare            │
├─────────────────────────────────┤
│ untitles.net → Vercel           │
│ api.untitles.net → AWS EC2      │
│ images.untitles.net → R2        │
└──────┬──────────────────────────┘
       │
       ▼
┌──────────┬──────────┬──────────┐
│  Vercel  │ AWS EC2  │    R2    │
│ 프론트엔드 │  백엔드   │  이미지   │
└──────────┴──────────┴──────────┘
```
1. 이미지 업로드 프론트 → EC2 → R2 저장 → DB에 URL 저장
2. 이미지 조회 프론트 → images.untitles.net (R2 직접)
