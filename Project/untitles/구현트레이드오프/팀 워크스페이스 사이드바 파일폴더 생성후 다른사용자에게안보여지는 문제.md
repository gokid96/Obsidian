사용자 1 과 사용자2가 같은 워크스페이스를 보고있음 
사용자 1 이 폴더랑 게시글을 만듦 
사용자 2에겐 업데이트가 안될텐데 이런 경우에는 어떡하지?

### 1. 주기적 폴링 (가장 간단)
```javascript
// 30초마다 트리 새로고침
setInterval(() => {
  folderStore.loadWorkspaceTree()
}, 30000)
```
**장점:** 구현 쉬움  
**단점:** 불필요한 API 호출, 실시간 아님

---

### 2. WebSocket / SSE (실시간)
```
사용자1 폴더 생성 → 서버 → 같은 워크스페이스 사용자들에게 이벤트 전송 → 사용자2 트리 업데이트
````

**장점:** 진짜 실시간  
**단점:** 서버 작업 필요 (Spring WebSocket 설정)

---

### 3. 포커스 시 새로고침 (절충안)
```javascript
// 브라우저 탭 다시 활성화될 때 새로고침
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'visible') {
    folderStore.loadWorkspaceTree()
  }
})
```

**장점:** 구현 쉬움, API 호출 적음  
**단점:** 탭 전환해야 반영됨
