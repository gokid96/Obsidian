
워크스페이스 모두 삭제 한 유저의 경우 어떻게 처리?

최소 1개 워크스페이스를 유지를 강제
## 방식 1: "내 노트" 삭제 불가

|장점|단점|
|---|---|
|사용자가 실수로 기본 공간을 잃지 않음|이름 변경하면? "내 노트2"는 삭제 가능?|
|항상 최소 1개 워크스페이스 보장|이름으로 판단하면 우회 가능|

---

## 방식 2: 마지막 1개 워크스페이스 삭제 불가

|장점|단점|
|---|---|
|이름에 의존하지 않음|"내 노트"가 아닌 워크스페이스만 남을 수 있음|
|더 유연함||

---


**방식 2 + 기본 워크스페이스 플래그** (UI에는 기본 워크스페이스인점 표시)

```java
// Workspace 엔티티에 추가
private boolean isDefault = false;  // 회원가입 시 생성된 워크스페이스

// 삭제 시 체크
if (workspace.isDefault()) {
    throw new IllegalArgumentException("기본 워크스페이스는 삭제할 수 없습니다.");
}
```

이렇게 하면:

- 이름 변경해도 삭제 불가
- 다른 워크스페이스는 자유롭게 삭제 가능

기본 워크스페이스는:

1. **삭제 불가**
2. **멤버 초대 불가** (혼자만 사용)

---

## 구현
```java
// Workspace 엔티티
private boolean isDefault = false;

// 삭제 시
if (workspace.isDefault()) {
    throw new IllegalArgumentException("기본 워크스페이스는 삭제할 수 없습니다.");
}

// 멤버 초대 시
if (workspace.isDefault()) {
    throw new IllegalArgumentException("기본 워크스페이스에는 멤버를 초대할 수 없습니다.");
}
```

## 프론트엔드

|항목|일반 워크스페이스|기본 워크스페이스|
|---|---|---|
|삭제 버튼|✅|❌ 숨김|
|멤버 탭|✅|❌ 숨김 또는 비활성화|
|이름 변경|✅|✅ 가능|
